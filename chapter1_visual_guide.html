Chapter 1 Visual Guide: Understanding Your CRM Data — data types, distributions, missing data, correlation, and dataset relationships

1 · Types of data in your CRM
Every column in your six Zoho tables falls into one of these types. The type determines how you process and encode it for ML.
📊
Continuous numerical no scaling yet
Real numbers — any value in a range including decimals. Arithmetic (mean, SD) is meaningful.
Visitor Score (348.5), Average Time Spent (61.8 min), Deposit Amount (2,000)
🔢
Discrete / count numerical whole numbers
Only whole numbers — counts of events. Cannot be 2.7 calls or 3.4 chats.
Number of Chats, Days Visited, No. of Follow-ups, Total Calls Made
🏷️
Nominal categorical one-hot encode
Labels with no natural order. Cannot say Referral > Website mathematically.
Language, Lead Source, Industry, Country
📶
Ordinal categorical ordinal encode 0,1,2…
Labels with a meaningful ranking. The model should know Interested > Prospect > New Lead.
Followup Status (New Lead → Prospect → Interested → Convert)
✅
Binary encode as 0 / 1
Exactly two possible values. Simplest encoding — directly usable.
Is Converted, Lapsed Client, Email Opt Out, Feedback 1 Done
📅
Date / time engineer numeric features
Raw timestamps are useless — extract gaps, hours, day-of-week as numbers.
Created Time → days_since_created, hours_to_first_call, created_on_weekend

3 · Missing data — what to do with each type
Each column's missing data has a different reason — and requires a different treatment. Never blindly fill all blanks with the mean.
Missing data rates — Followup table
Each bar = % of rows where this column is blank
Three types of missingness
MCAR — Missing Completely At Random
Salesperson forgot to fill it in. No pattern.
Fix: Fill with median (numbers) or mode (categories).
MNAR — Missing Not At Random
Visitor Score blank = lead never visited the site. True value is 0.
Fix: Fill with 0 — the correct sentinel value.
Informative Missingness
Whether blank or filled is itself the signal. Deposit Amount blank = never deposited.
Fix: Create has_deposit = 0/1 flag BEFORE filling with 0.
